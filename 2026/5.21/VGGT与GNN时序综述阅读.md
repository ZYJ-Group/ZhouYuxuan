## 二、文献一：VGGT

### 2.1 研究背景

​	传统三维重建一般依赖 SfM、MVS和BA 等视觉几何方法，这些方法，整体流程较复杂，计算代价也较高。

​	于是深度学习开始参与三维视觉任务。例如 DUSt3R、MASt3R 等方法已经显示出有希望的结果，但这类方法在多图像场景下仍然常常依赖后处理优化或两两图像融合。VGGT 的出发点是进一步减少对传统几何优化流程的依赖，让神经网络通过一次前向推理直接完成多个三维视觉任务。

### 2.2 核心思想

VGGT 的核心思想：

​	使用一个大规模前馈式 Transformer，在较少人工三维归纳偏置的前提下，通过大量三维标注数据训练，使模型直接学习从多视角图像到三维场景属性之间的映射。

模型输入是一组观察同一场景的 RGB 图像，输出包括以下四类信息：

| 输出内容 | 含义 |
| --- | --- |
| 相机参数 | 相机内参和外参，用于描述每张图像的拍摄位姿与成像参数 |
| 深度图 | 每张图像对应的像素级深度信息 |
| 深度图 | 每个像素对应的三维点坐标 |
| 点跟踪 | 不同图像之间同一三维点的二维对应轨迹 |

​	它不是只做一个任务，而是把相机估计、深度估计、三维重建和点跟踪放在一个模型里一起学。

### 2.3 方法结构

VGGT 的整体流程可以理解为：

```text
多张输入图像
→ DINO 提取图像 token
→ 加入 camera token
→ Transformer 进行多视角信息融合
→ 不同预测头输出相机、深度、点图和跟踪结果
```

​	其中，DINO 负责将输入图像转化为视觉 token；主干网络由多层 Transformer 组成；预测头则分别完成相机参数、深度图、点图和点跟踪结果输出。

​	其中比较关键的是替注意力机制。它一会儿在单张图像内部建模，一会儿在所有图像之间建模。这样既能看清单张图像内部的信息，也能把多视角之间的关系联系起来。

### 2.4 实验结果

​	论文在多个三维视觉任务上验证了 VGGT 的有效性，包括相机位姿估计、多视角深度估计、点图估计、双目图像匹配、动态点跟踪和新视角合成。整体结论是：VGGT 在很多任务上不仅速度快，而且精度也很高。更重要的是，它即使不做复杂后处理，也能得到比较好的三维结果；如果再结合 BA 优化，效果还可以继续提升。

​	这说明模型学习到的并不只是某一类数据集的局部规律，而是具有一定通用性的视觉几何表征。

## 三、文献二：GNN 时间序列综述

### 3.1 研究背景

​	时间序列数据广泛存在于交通、能源、金融、物联网、工业系统和医疗等领域。传统时间序列方法，如 ARIMA、VAR、SVR 等，在处理复杂非线性关系和多变量关系时能力有限。虽然 CNN、RNN、Transformer 等深度学习方法能够建模时间依赖，但它们通常不能显式表达不同变量之间的结构关系。

​	图神经网络 GNN 可以把不同变量、传感器或区域看作图中的节点，把它们之间的距离、连接、相似性或因果关系看作边，从而同时建模时间依赖关系和变量之间的空间关系。因此，GNN 很适合处理复杂的多变量时间序列问题。

​	图神经网络在时间序列分析中的应用，主要包括预测、分类、缺失值填补和异常检测四类任务

### 3.2 基本概念理解

​	这篇综述首先区分了单变量时间序列和多变量时间序列。对于 GNN4TS 来说，多变量时间序列更常见，因为多个变量可以自然抽象为图中的节点。

图结构的构建方式主要有两类：

| 类型           | 含义                                       |
| -------------- | ------------------------------------------ |
| 人工规则构图   | 根据距离、连接关系、相似度、因果关系等构图 |
| 模型自动学习图 | 让模型从数据中自己学出变量之间的关系       |

​	启发式图结构可解释性较强，但可能遗漏潜在关系；学习式图结构更灵活，但可解释性和稳定性需要进一步验证。

### 3.3 GNN4TS 的四类任务

综述将 GNN4TS 的主流任务归纳为四类：

| 任务             | 作用                 | 例子                         |
| ---------------- | -------------------- | ---------------------------- |
| 时间序列预测     | 根据历史数据预测未来 | 交通流量、电力负荷、风速预测 |
| 时间序列分类     | 判断一段序列属于哪类 | 行为识别、故障分类           |
| 时间序列补全     | 填补缺失数据         | 传感器数据修复               |
| 时间序列异常检测 | 找出不正常的变化     | 工业故障、网络异常           |

### 3.4 方法结构

一个典型的 GNN 时序模型通常包含两部分：

| 模块     | 作用                       |
| -------- | -------------------------- |
| 空间模块 | 建模不同变量之间的关系     |
| 时间模块 | 建模同一个变量随时间的变化 |

​	空间模块常用 GCN、GAT、消息传递等方法；时间模块常用 RNN、GRU、TCN、Transformer 等方法。

### 3.5 理解

​	这篇综述让我对 GNN4TS 的整体研究框架有了较完整的认识。很多时间序列并不是孤立存在的，而是来自一个多实体系统。只看单条序列的时间变化是不够的，还必须考虑变量之间的关系。
